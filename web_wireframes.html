tameru · web wireframes · v2 (persistent composer)

Updated to the persistent-bottom-composer pattern. The chat entry point is now always visible at the bottom of every core screen — no sidebar button, no "click to open." Just type. The placeholder rotates through real example prompts every few seconds (try waiting and watching) so new users learn what they can ask.

On submit, a 440px right drawer slides in without dimming the rest of the page — the dashboard stays visible and clickable. Frame 15 shows that end state. The composer "morphs" from bottom-center into the drawer's bottom; static frames can't show the animation, so think of it as one element that relocates.

Frames at 1280×760 (production: 1440×900). Full spec in UX_WEB_PROMPT.md.
